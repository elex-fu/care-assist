## 一、产品介绍

### 1.1 产品定位

**产品名称**：家庭智能健康助手

**一句话描述**：拍一拍报告、问一问AI、看一看趋势、管一管全家健康。

**核心定位**：以家庭为单位、以AI为核心驱动、覆盖门诊/住院/慢病/儿童全场景的健康管理智能平台。

### 1.2 目标用户

| 用户角色 | 特征 | 核心需求 |
|---------|------|---------|
| **家庭健康管理者**（30-50岁） | 上有老下有小，管理全家健康 | 一站式管理多人健康，快速了解异常，获得行动建议 |
| **慢性病患者/家属** | 高血压/糖尿病/高血脂长期随访 | 指标趋势追踪、用药管理、复查提醒、病情变化预警 |
| **住院患者家属** | 亲人在院，需持续关注指标变化 | 住院指标实时盯盘、日对比、医生沟通准备 |
| **新手父母** | 0-6岁儿童疫苗、成长发育管理 | 疫苗排程与提醒、生长曲线评估、发育里程碑追踪 |

### 1.3 核心价值

1. **降低信息门槛** — 拍照上传报告，AI自动提取指标、解读异常
2. **趋势即洞察** — 指标纵向对比，发现变化趋势，预警健康风险
3. **AI健康顾问** — 随时可问的病情咨询、医生沟通建议、生活注意事项
4. **全家健康管理** — 一个平台管理多成员，差异化展示，成员间数据共享

### 1.4 核心设计原则

| 原则 | 含义 |
|------|------|
| **One Way, Everywhere** | 同一功能只有一个入口、一种交互、一种视觉 |
| **Context Flow** | 用户永远知道"我在哪、从哪来、能去哪" |
| **Closed Loop** | 每个行为有完整闭环，终点是下一个起点 |
| **三次点击原则** | 核心路径≤3次点击，信息直接显性 |
| **先结论后细节** | 摘要先行，详情折叠，每次只给最需要的信息 |

### 2.2 核心功能模块详述

#### 模块1：首页 — 家庭看板

**布局**：纵向滚动，顶部家庭名称 + 搜索图标 + AI图标 → AI健康日报横幅 → 成员卡片网格（2列）→ 底部5Tab导航。

**成员卡片差异化显示**：

| 成员类型 | 卡片摘要内容 | 角标 |
|---------|------------|------|
| 成人慢病 | 关注指标最新值 + 趋势箭头（如"血压128/82 ↓12"） | 异常时红色条 |
| 成人健康 | 最近一次体检日期 | 无 |
| 儿童 | 疫苗进度% + 下次接种日期 | 疫苗到期⚠ |
| 住院中 | 住院天数 + 今日检查数 + 异常数 | "住院中"紫色Badge |

**AI入口**：顶部导航栏AI图标、每张成员卡片"问AI"按钮、AI健康日报横幅、底部导航中央AI按钮。

#### 模块2：个人档案 — 时间轴

**布局**：顶部成员信息 → 筛选标签 → 纵向时间轴。

**时间轴节点类型与颜色**：

| 类型 | 色条/圆点 | 图标 |
|------|----------|------|
| 就诊 | #3B82F6 蓝 | 🏥 |
| 检验 | #10B981 绿 | 📋 |
| 用药 | #F59E0B 橙 | 💊 |
| 症状 | #94A3B8 灰 | 🤒 |
| AI解读 | #8B5CF6 紫 | 🤖 |
| 住院 | #8B5CF6 紫 | 🏥 |
| 疫苗 | #10B981 绿 | 💉 |
| 里程碑 | #F59E0B 橙 | 🧸 |

**住院节点特殊设计**：紫色背景条跨越住院天数，比常规节点高40px，显示入院日期-出院日期、医院、科室、住院天数，点击展开住院摘要浮层。

#### 模块3：指标中心

**两种视图模式**：
- **列表视图**：指标卡片（名称 + 最新值 + 单位 + 趋势箭头 + 状态Badge），点击→趋势图Bottom Sheet，右上角⋮菜单→复制/问AI/查看趋势/分享。
- **表单视图**：日期×指标透视表，行=日期，列=指标名称，异常数值红色文字+浅红背景，横向滚动查看所有指标。

**慢性病套餐**：高血压（收缩压+舒张压+心率+用药）、糖尿病（空腹血糖+餐后血糖+糖化血红蛋白+用药）、高血脂（总胆固醇+甘油三酯+LDL+HDL），多指标叠加趋势图+AI管理建议。

#### 模块4：住院专页

**四层递进视图**（顶部Tab：总览/盯盘/对比/批次）：

| 层级 | 回答的问题 | 核心交互 |
|------|-----------|---------|
| **总览** | 住院全貌？哪天手术？每天查了什么？ | 每日检查批次卡片纵向排列 |
| **盯盘** | 最关键指标现在怎么样？异常几个？ | 3列卡片网格，自选任意数量 |
| **对比** | 今天比昨天好了吗？ | 任意两天对比，自动计算变化+评价 |
| **批次** | 某天某次检查的所有指标详情？ | 灵活分组检查单，异常置顶 |

**长期住院聚合**：按天(1-30天) / 按周(30-90天) / 按月(90天以上)，显示均值+范围+异常天数。

#### 模块5：AI助手

**三层入口体系**：

| 层级 | 位置 | 形式 |
|------|------|------|
| 全局常驻 | 底部导航中央 + 右下角悬浮球 | 紫色AI按钮（凸起）+ 可拖拽悬浮球 |
| 页面级 | 每个信息块 | 问AI按钮 / ⋮菜单中的问AI / 长按菜单 |
| 主动推送 | 系统通知 | 异常预警 / 复查提醒 / 每日简报 |

**AI对话浮层**：半屏(60%)：上下文栏 + 快捷问题2×2网格 + 输入栏；全屏(100%)：完整对话 + 数据卡片 + 追问推荐；上滑展开全屏，下滑关闭。

**AI回答规范（5层结构）**：
1. 直接回答（2-3句白话）
2. 数据支撑（引用指标数值+日期）
3. 可点击数据卡片（点击→趋势图）
4. 建议/下一步行动
5. 追问推荐（2-4个相关问题）

#### 模块6：上传流程（3步极简）

```
Step1: 拍照（默认唤起相机，相册/手动收进"更多"）
       ↓
Step2: AI多模态识别（全屏loading动画）
       后台自动完成：识别成员（姓名/年龄匹配）→ 提取指标 → 判断异常 → 自动分类报告类型 → 关联住院事件（如有）
       ↓
Step3: 结果页
       ├── 一句话AI总结："已记录！发现2项需要注意"
       ├── 异常项人话解读："血压偏高，建议下周复查"
       └── 唯一行动按钮："好的，提醒我复查"
       └── （数据不对？提供"点击修正"非强制入口）
```

#### 模块7：异常→行动闭环

```
发现异常
  ↓
异常标识（三层体系：色条+数值变色+Badge）
  ↓
点击异常 → 展开异常详情Bottom Sheet
  ↓
  ├── AI白话解读（异常含义+可能原因）
  ├── 数据支撑（历史趋势缩略图）
  └── 两级行动建议卡片（后端根据偏离程度自动判断）：
      ├── 🟢 先观察（轻微异常，建议1-2周后复查观察）
      └── 🔴 需要去看看（偏离较大或趋势恶化，建议3天内复查或尽快就医）
  ↓
选择行动：
  ├── 🟢 观察 → 添加到提醒（7天后）
  └── 🔴 去看看 → 后端自动判断：偏离20-30%则3天内复查，>30%或恶化则尽快就医 + 生成就诊摘要PDF
  ↓
复查提醒到期 → 标记"已完成" → 引导上传复查报告 → AI自动对比新旧数据
```

## 三、UI交互设计

```
│  首页   │  指标   │  🤖 AI  │  上传   │  我的   │
```

### 4.4 AI Provider 抽象层

**设计目标**：所有 AI 能力通过统一接口调用，不同 Provider（Kimi、DeepSeek、OpenAI、通义千问等）实现同一接口，通过配置切换，无需改动业务代码。

```python
class AIProvider(ABC):
    @abstractmethod
    async def chat(self, messages: list, stream: bool = False) -> AsyncIterator[str] | str:
        """通用对话接口，支持流式与非流式"""

    @abstractmethod
    async def analyze_image(self, image_url: str, prompt: str) -> str:
        """图片分析接口（报告 OCR / 指标提取）"""

    @abstractmethod
    async def generate_summary(self, context: dict) -> str:
        """生成就诊摘要 / 病情简报"""
```

**Provider 实现**：

| Provider | 实现类 | 说明 |
|----------|--------|------|
| Kimi (Moonshot) | `KimiProvider` | 长上下文、多模态视觉 |
| DeepSeek | `DeepSeekProvider` | 高性价比、快速响应 |
| OpenAI | `OpenAIProvider` | 国际备用 |
| 通义千问 | `QwenProvider` | 阿里系备用 |

**切换策略**：
- 默认 Provider 通过环境变量配置
- 单 Provider 故障时自动降级到备用 Provider
- 新增 Provider 只需实现 `AIProvider` 接口，零业务代码侵入

**通信协议分工**：
- **HTTP REST API**：负责所有功能型请求（成员管理、指标CRUD、报告上传、时间轴查询、提醒管理等），无状态、可缓存、易扩展。

### 5.1 异常判断算法

```typescript
interface IndicatorThreshold {
  name: string;
  unit: string;
  lowerLimit: number | null;
  upperLimit: number | null;
  ageGroups?: {
    minAge: number;        // 月龄
    maxAge: number;
    lowerLimit: number;
    upperLimit: number;
  }[];
}

// 核心判断函数
function judgeAbnormal(
  value: number,
  threshold: IndicatorThreshold,
  ageInMonths?: number
): 'normal' | 'low' | 'high' | 'critical' {
  let lower = threshold.lowerLimit;
  let upper = threshold.upperLimit;

  // 儿童按年龄段调整阈值
  if (threshold.ageGroups && ageInMonths !== undefined) {
    const group = threshold.ageGroups.find(
      g => ageInMonths >= g.minAge && ageInMonths <= g.maxAge
    );
    if (group) {
      lower = group.lowerLimit;
      upper = group.upperLimit;
    }
  }

  if (lower !== null && value < lower * 0.7) return 'critical';
  if (upper !== null && value > upper * 1.3) return 'critical';
  if (lower !== null && value < lower) return 'low';
  if (upper !== null && value > upper) return 'high';
  return 'normal';
}
```

**关键指标阈值示例**：
- 收缩压：90-140 mmHg
- 血红蛋白：按年龄段区分（新生儿100-140，婴儿110-145，成人120-160，老人110-160）
- 空腹血糖：3.9-6.1 mmol/L
- 白细胞：4-10 10^9/L

### 5.2 趋势评价算法

```typescript
function evaluateTrend(
  current: number,
  previous: number,
  threshold: IndicatorThreshold,
  isLowerBetter: boolean
): { direction: string; magnitude: string; evaluation: string } {
  const change = current - previous;
  const changePercent = Math.abs(change / previous);

  // 方向
  let direction: string;
  if (Math.abs(changePercent) < 0.05) direction = 'stable';
  else if (change > 0) direction = 'up';
  else direction = 'down';

  // 幅度
  let magnitude: string;
  if (changePercent < 0.1) magnitude = 'small';
  else if (changePercent < 0.3) magnitude = 'moderate';
  else magnitude = 'large';

  // 评价（结合指标特性：越低越好如血糖，越高越好如血红蛋白）
  let evaluation: string;
  if (isLowerBetter) {
    evaluation = direction === 'down' ? 'improving' : direction === 'up' ? 'worsening' : 'stable';
  } else {
    evaluation = direction === 'up' ? 'improving' : direction === 'down' ? 'worsening' : 'stable';
  }

  // 持续异常
  const status = judgeAbnormal(current, threshold);
  if (status !== 'normal' && magnitude === 'small') {
    evaluation = 'concerning';
  }

  return { direction, magnitude, evaluation };
}
```

| 指标中心加载 | < 2秒 | 虚拟列表+懒加载 |
| 趋势图渲染 | < 1秒 | 数据采样(>100点降采样) |
| AI对话响应 | < 5秒 | "思考中"动画，超时切换模型 |
| OCR识别 | < 10秒 | 进度条，后台识别 |
| 小程序包体积 | 主包<2MB | 分包加载+图片CDN |
| 内存占用 | < 150MB | 图片缓存上限50MB |
| 列表滚动 | 60fps | 虚拟列表(>50条时) |

### 5.9 安全与合规

| 措施 | 实现方式 |
|------|---------|
| 传输加密 | HTTPS全站TLS 1.3 |
| 存储加密 | 服务器端AES-256加密敏感字段 |
| 图片安全 | 云存储私有Bucket，通过签名URL访问，7天过期 |
| 访问控制 | 成员只能访问自己家庭的数据 |
| 审计日志 | 所有数据操作记录日志（who/when/what） |
| 医疗免责声明 | 首页+AI对话底部常驻："本应用仅供参考，不构成医疗建议。如有不适请及时就医。" |
| 隐私协议 | 首次使用需同意《隐私协议》+《用户协议》 |
| 儿童隐私 | 16岁以下用户数据加密存储，分享需家长确认 |
| 数据删除 | 用户可导出全部数据，可申请删除账号（30天冷静期后物理清除） |
| 数据保留 | AI对话90天/审计日志1年/报告原图账号注销后1年/软删除成员1年后物理清理 |

## 六、项目实施建议

### 6.1 开发阶段规划

```
Phase 1（基础设施，2周）：
  - 后端：FastAPI 项目骨架、MySQL 表结构设计、Redis 连接池、Celery 任务队列
  - 前端：原生微信小程序项目搭建、公共组件（卡片/趋势图/Bottom Sheet）封装
  - DevOps：CI/CD 流水线、对象存储、测试环境部署

Phase 2（MVP核心，4-5周）：
  - 微信小程序端：成员管理/指标中心/上传OCR/AI对话
  - 后端：HTTP REST API 全量接口、JWT 鉴权、异常判断算法
  - AI：AIProvider 抽象层实现、WebSocket 流式对话接入、Celery 异步 OCR

Phase 3（场景扩展，3周）：
  - 住院场景（总览/盯盘/对比/批次）
  - 儿童专区（疫苗/成长曲线/里程碑）
  - 提醒与日历、数据导出

Phase 4（打磨优化，2周）：
  - 测试覆盖（单元测试≥80%、核心算法100%边界覆盖、AI OCR黄金集回归、Minium E2E核心路径）
  - 性能调优（MySQL 索引/Redis 缓存/接口压测）
  - 边界情况处理（无网络/弱网/大量数据）
  - 安全加固（SQL 注入防护/XSS 过滤/敏感数据加密）
  - 长辈模式适配、大字体模式、色盲友好模式
```

### 6.3 后端语言选型说明（Python vs Go）

本项目最终选用 **Python** 作为后端语言，理由如下：

| 维度 | Python | Go | 本项目倾向 |
|------|--------|-----|-----------|
| **AI 生态** | 碾压优势：OpenAI SDK、LangChain、各厂商 LLM SDK 均为 Python 优先 | 生态薄弱，大多需自行封装 HTTP 客户端 | **Python**（核心差异化是 AI 能力） |
| **Prompt 工程** | Jupyter/iPython 交互式调试，迭代效率极高 | 无交互式环境，调试需重新编译运行 | **Python**（Prompt 需持续调优） |
| **数据处理** | pandas/numpy 做指标分析、趋势计算一行代码搞定 | 需手写算法，开发效率低 | **Python**（指标分析是核心逻辑） |
| **图像处理** | Pillow/opencv-python 做 OCR 预处理成熟稳定 | 需 cgo 调用或外部服务 | **Python**（报告图片处理频繁） |
| **开发速度** | 语法简洁，MVP 验证快 | 需处理大量 boilerplate | **Python**（快速迭代优先） |
| **异步性能** | FastAPI + asyncio 完全满足本项目并发（IO 密集型） | 原生并发更强，但本项目非计算密集型 | 平手（AI 调用本身是 IO 等待） |
| **部署运维** | 需处理依赖/虚拟环境，Docker 化后差异不大 | 单二进制部署更轻量 | Go 略胜（可通过 Docker 抹平） |

**结论**：本项目是 AI 驱动型应用，核心工作负载是 **AI API 调用编排 + 数据处理 + Prompt 工程**，而非高并发计算。Python 在这些维度的开发效率和生态成熟度显著优于 Go，因此选择 Python + FastAPI。

### 6.2 关键成功因素

1. **AI识别准确率**：报告OCR和指标提取的准确率直接影响用户体验，需要持续优化Prompt和校验流程。
2. **异常判断标准**：医学指标参考范围必须权威准确，儿童年龄段阈值需严格依据国家标准。
3. **数据安全**：健康数据属于敏感个人信息，必须确保传输加密、存储加密、访问控制三重保障。
4. **闭环体验**：从发现异常→理解异常→采取行动→验证结果，每个环节都要顺畅衔接。

## 七、功能设计深度优化建议（面向小白用户）

### 7.1 核心问题诊断

当前方案功能覆盖全面，但对小白用户存在以下认知负担：

| 问题 | 具体表现 | 小白用户的影响 |
|------|---------|--------------|
| **功能过载** | 儿童13个子模块、住院4层视图、2种指标视图 | 打开页面不知所措，找不到重点 |
| **术语生僻** | "盯盘"、"批次"、"表单视图"、"透视表" | 需要学习成本，产生畏惧感 |
| **入口分散** | AI有三层入口、上传有多种方式、视图可切换 | 不知道点哪里是对的 |
| **流程过长** | 上传5步、共享需验证等待确认 | 中途容易放弃 |
| **缺少引导** | 首次打开无 step-by-step 引导 | 第一步该做什么完全不知道 |
| **判断标准技术化** | "偏离10%"、"偏离30%" | 用户不关心数字，只关心"要不要去医院" |

### 7.2 优化原则："傻瓜式"设计

**核心原则：用户不需要学习，打开就会用。**

```
用户视角的简化逻辑：
"我家人的健康怎么样？" → 首页一眼看到
"有没有问题？" → 红色=有问题，绿色=没问题
"有问题怎么办？" → 点击红色 → AI告诉你怎么办
"怎么记录新报告？" → 点+号 → 拍照 → 完成
```

### 7.3 具体优化方案

#### 优化1：首页极简化 — "一眼看全家的健康红绿灯"

**问题**：当前首页有AI日报横幅+成员卡片+5Tab，信息密度高。

**优化后**：
```
┌──────────────────────────────┐
│ 家庭健康          [+]        │  ← 右上角快捷"+"上传按钮
├──────────────────────────────┤
│ 🤖 AI健康日报横幅（可展开）   │  ← 一行摘要+展开详情
├──────────────────────────────┤
│                              │
│   [爸爸]        [妈妈]       │  ← 成员卡片显示：头像+名字+红绿灯摘要
│   🟢 血压128/82 ↓12        │    慢病成员：关注指标最新值+趋势
│   🟢 健康                  │    健康成员：最近体检日期
│   [小宝]        [奶奶]       │    儿童：疫苗进度%+下次接种
│   🟡 疫苗85%  🔴 住院中第5天 │    住院：住院天数+今日检查数+异常数
│                              │
├──────────────────────────────┤
│ 🏠  指标  🤖  上传  我的    │  ← 底部5Tab：首页/指标/AI/上传/我的
└──────────────────────────────┘
```

**关键改动**：
- 保留AI日报横幅（小白用户可折叠，一行摘要+点击展开详情，不强制阅读）
- 成员卡片保留**差异化摘要**（慢病成员显示指标值+趋势，儿童显示疫苗进度，住院显示天数）
- 底部Tab保持**5个**：「首页」「指标」「AI」「上传」「我的」
  - 「首页」= 家庭看板，一眼看全家状态
  - 「指标」= 快捷入口，直达关注指标趋势
  - 「AI」= AI对话独立页面
  - 「上传」= 拍照上传报告
  - 「我的」= 设置+成员管理
- AI保留**全局悬浮球**作为辅助快捷入口（右下角，与底部AI Tab并存，悬浮球随时可唤起，不占屏幕主体空间）
- 首页右上角保留**"+"号**作为快捷上传入口（与底部上传Tab并存，首页拍照更方便）

#### 优化2：上传流程 — "拍照即完成"

**问题**：旧版5步流程（选方式→选成员→AI识别→确认→入库）步骤过多，小白在中途会困惑。

**优化后：3步极简流程**

```
Step1: 点击"+" → 直接唤起相机（默认拍照，相册/手动藏在二级菜单）
        ↓
Step2: 拍照完成 → 全屏loading动画："AI正在读报告..."
        ↓  （后台自动识别成员（通过报告上的姓名/年龄）、自动提取指标、自动判断异常）
Step3: 结果页 → "已记录！发现2项异常"
        ├── 直接显示AI一句话解读："血压偏高，建议下周复查"
        ├── 只有1个按钮："好的，提醒我复查"
        └── （无需确认表格、无需选择报告类型、无需关联住院——全部AI自动完成，错误再手动修正）
```

**关键改动**：
- 去掉"选择方式"步骤，默认直接拍照，相册/手动收进"更多"
- 去掉"选择成员"步骤，AI通过报告上的姓名/年龄自动匹配（不匹配再手动选）
- 去掉"确认提取结果"表格，AI自动入库，只在结果页用**一句话总结**告诉用户发生了什么
- 错误修正不强制，提供"数据不对？点击修正"的入口（非强制步骤）
- 报告类型（检验/诊断/处方/出院）由AI自动判断，无需用户选择

#### 优化3：异常引导 — 不用看懂数字，只用看懂颜色

**问题**：当前三级行动用"偏离10%""偏离30%"判断，小白用户需要理解百分比含义。

**优化后：红绿灯+人话**

```
点击红色异常 → 展开Bottom Sheet

┌──────────────────────────────┐
│ 血压 145/90 mmHg             │
│ 🟡 比正常高一点点             │  ← 不用"偏高"，用"高了一点点"
├──────────────────────────────┤
│ 🤖 AI说：                     │
│ "血压比正常高了15，不算严重， │
│  但建议下周去社区医院复查一下" │  ← 只说人话，不说医学术语
├──────────────────────────────┤
│ [🔔 帮我定个复查提醒]          │  ← 只有一个行动按钮
└──────────────────────────────┘
```

**关键改动**：
- 三级行动合并为**两级**：「没事，观察一下」「需要去看看」
- 去掉具体偏离百分比，用口语化描述："高了一点点""低了一些""差得有点多"
- 每个异常只给**一个推荐行动按钮**（不是三个让用户选）
- AI解读只说**人话**，不出现医学术语（如用"血里的红细胞"代替"血红蛋白"）

#### 优化4：儿童专区 — "需要做什么，就显示什么"

**问题**：当前13个子模块全部展示，新手父母一看就懵。

**优化后：按龄动态显示**

```
孩子3个月大时看到的页面：
┌──────────────────────────────┐
│ 小宝（3个月）                 │
├──────────────────────────────┤
│ 🔴 本月要做的事：              │  ← 只显示当前年龄段该做的事
│ 1. 百白破疫苗（第2针）5/15到期 │
│ 2. 3月龄体检                  │
├──────────────────────────────┤
│ 🟢 最近挺好的：                │  ← 正面反馈，缓解焦虑
│ · 身高体重正常                │
│ · 已学会抬头                  │
├──────────────────────────────┤
│ [📈 成长记录] [💉 疫苗]        │  ← 只有2个入口，不是13个
│ [🤖 问AI]                     │
└──────────────────────────────┘
```

**关键改动**：
- 13个子模块**不全部展示**，改为"按孩子当前月龄动态推荐"
- 页面只分**3个区块**：「本月要做的事」（待办）、「最近挺好的」（已完成/正常）、「快捷入口」（成长记录/疫苗/问AI）
- 疫苗、成长曲线、里程碑等作为**二级页面**，不是一级入口
- 用「待办清单」思维替代「功能菜单」思维

#### 优化5：住院场景 — "今天比昨天好了吗？"

**问题**：当前4层视图（总览/盯盘/对比/批次），小白家属不知道先点哪个。

**优化后：3个Tab，名字直白**

```
住院页面顶部Tab改为：
[每天的情况] [关键指标] [对比变化]

而不是：
[总览] [盯盘] [对比] [批次]
```

**关键改动**：
- 「总览」+「批次」合并为「每天的情况」：按日期排列，每天一张卡片，显示"查了血常规、拍了CT"，点击展开当天详情
- 「盯盘」改为「关键指标」：只显示医生/AI推荐的5-8个关键指标，不是让用户自选（小白不知道选什么）
- 「对比」改为「对比变化」：默认"今天vs昨天"，不需要用户选择日期，一打开就看到变化
- 去掉"批次"概念，用户不关心"批次"，只关心"今天查了什么"

#### 优化6：AI助手 — "一个按钮解决所有问题"

**问题**：当前AI有三层入口（底部Tab/悬浮球/页面级问AI按钮），小白不知道有什么区别。

**优化后：只有一个AI入口**

```
全局只有一个AI入口：右下角悬浮AI球（紫色，带sparkle图标）

点击后展开：
┌──────────────────────────────┐
│ 🤖 有什么想问的？              │
├──────────────────────────────┤
│ [爸爸血压高怎么办？]           │  ← 快捷问题只有2个，根据当前状态动态
│ [小宝下次疫苗什么时候？]       │
├──────────────────────────────┤
│ [输入问题...]        [发送]    │
└──────────────────────────────┘
```

**关键改动**：
- 底部Tab保留独立的"AI"Tab，同时保留**全局悬浮球**作为快捷入口（双入口并存：Tab适合深度对话，悬浮球适合随时快速提问）
- 页面内"问AI"按钮保留（成员卡片/指标详情/报告页），但统一唤起同一AI对话面板，避免入口歧义
- 快捷问题从4个减到**2个**，且必须是最相关的（如爸爸有异常时只显示"爸爸血压高怎么办？"）
- AI回答必须先给**结论**（"没事"或"建议复查"），再给解释

#### 优化7：首次使用引导 — "手把手教第一步"

**问题**：当前方案没有设计首次使用引导，小白打开App不知道做什么。

**优化后：4步强制引导（+1屏欢迎）**

```
欢迎屏："欢迎来到家庭健康助手"
        "4步帮你管好全家健康"
        [开始] →

第1屏："第1步：添加第一位家人"
        直接弹出添加成员表单（预填"爸爸/妈妈/孩子"选项）
        只需填：名字 + 生日 + 关系（3个字段）
        → 完成

第2屏："第2步：拍一张最近的体检报告"
        直接唤起相机（不给选择，直接拍）
        没有报告？提供[手动输入一个指标]作为替代演示路径
        拍完后显示："AI正在读报告..."
        → 完成后显示："已记录！"

第3屏："第3步：看看AI怎么说"
        直接展示AI解读结果
        "爸爸血压偏高，建议下周复查"
        [🔔 帮我定个提醒]

第4屏："第4步：开启健康提醒"（可选但推荐）
        引导用户授权微信订阅消息：
        - 「每日健康摘要」每天早上8点汇总待办
        - 「紧急提醒」严重异常/疫苗逾期时立即通知
        用户可勾选授权，也可跳过

完成 → 进入首页，看到爸爸卡片是🟡黄色
```

**关键改动**：
- 首次使用不展示空首页，而是**强制引导**（添加成员→拍照→看AI解读→开启提醒）
- 添加成员表单极简：只保留**名字+生日+关系**，头像/血型/过敏史等收进"更多信息"
- 引导完成后，用户首页已经有**第一个成员+第一份报告+第一个AI解读**，不是空页面
- 订阅授权放在最后一步，此时用户已感受到产品价值，授权意愿更高

#### 优化8：家庭成员共享 — "像微信拉群一样简单"

**问题**：当前共享流程：创建者生成邀请码→被邀请人填写姓名关系→等待确认→创建者确认。对小白来说太复杂。

**优化后：一键拉群**

```
创建者：
点击"邀请家人" → 直接调起微信分享 → 分享到家庭群

被邀请人：
点击分享卡片 → 一键授权登录 → 自动加入家庭
（无需填写姓名关系，微信昵称直接作为默认名，关系可选）

完成。没有"等待确认"步骤，默认信任微信好友关系。
```

**关键改动**：
- 去掉邀请码，只用**微信分享卡片**
- 去掉"验证身份"步骤，被邀请人点击分享卡片即加入
- 去掉"创建者确认"步骤，默认信任（后续可移除）
- 姓名默认取微信昵称，关系默认"家人"（可修改）

#### 优化9：提醒机制 — "不打扰，只提醒该提醒的"

**问题**：当前提醒类型多（疫苗/复查/体检/用药），推送频繁会打扰用户。

**微信小程序订阅消息限制**：
微信规定，小程序必须通过用户**主动点击订阅按钮**授权后，才能发送订阅消息。用户未授权时，无法静默推送。这要求我们在产品流程中设计自然的授权引导，而非假设推送一定能到达。

**优化后：智能合并推送 + 分场景授权引导**

```
已订阅用户：每天早上8:00收到一条「今日待办」
"今天有3件事：
  1. 爸爸该复查血压了
  2. 小宝的疫苗5天后到期
  3. 奶奶住院第5天，今天有新的检查报告"

未订阅用户：打开小程序时首页顶部显示「今日待办」横幅
  + 关键事件（严重异常/疫苗逾期）在应用内红点亮起
```

**关键改动**：
- 同一天的多个提醒**合并为一条**「今日待办」推送，减少打扰
- 只有**真正紧急的事**才单独推送（如指标严重异常、疫苗逾期）
- **订阅授权引导**嵌入自然场景：
  - 首次引导最后一步：引导订阅「每日待办」+「紧急提醒」两个模板
  - 每次设置提醒时：若未订阅对应模板，先弹出友好授权申请
  - 上传发现异常后：在结果页设置复查提醒时，引导订阅「复查提醒」模板
- **未订阅兜底**：WebSocket实时推送（在线时）+ 首页「今日待办」横幅（离线后打开时）
- 用户可在「我的→提醒设置」中管理订阅状态

#### 优化10：时间轴 — "只显示用户关心的"

**问题**：当前时间轴有8种节点类型（就诊/检验/用药/症状/AI/住院/疫苗/里程碑），颜色和图标各异，小白需要记忆。

**优化后：3种颜色**

```
时间轴只分3种：
🟢 正常记录（体检、疫苗接种、正常指标）——绿色
🟡 需要注意（轻微异常、复查提醒、疫苗快到期）——黄色
🔴 重要事件（严重异常、住院、急诊）——红色

去掉：用药节点（收进详情）、症状节点（收进详情）、AI解读节点（收进详情）
```

**关键改动**：
- 时间轴节点类型从8种减到**3种**（正常/注意/重要），只按严重程度分色
- 用药、症状、AI解读等辅助信息收进**详情页内**，不在时间轴上分散展示
- 用户不需要记忆"蓝色=就诊、绿色=检验"，只需要记忆"红色=有问题"

### 7.4 优化后的信息架构（5Tab完整版）

```
家庭智能健康助手
│
├── 首页（家庭看板）
│   ├── AI健康日报横幅（一行摘要+可展开）
│   ├── 成员卡片网格（2列，差异化摘要+红绿灯）
│   └── 快捷操作栏
│
├── 指标（快捷入口）
│   └── 关注指标趋势（直达个人指标中心）
│
├── AI（独立页面+悬浮球快捷入口）

│   ├── 全屏浮层 — 深度对话 + 数据卡片
│   └── 主动推送 — 异常/复查/日报通知
│
├── 上传（流程化，3步极简）
│   ├── Step1: 拍照（默认）/相册/手动
│   ├── Step2: AI识别中（loading动画）
```

### 7.5 优化前后对比

| 维度 | 优化前 | 优化后 | 小白用户收益 |
|------|--------|--------|-------------|
| 底部Tab | 5个 | **5个**（保留完整导航） | 功能入口清晰，不隐藏 |
| 儿童模块 | 13个一级入口 | **3个区块**（待办/正常/快捷入口） | 一眼知道该做什么 |
| 住院Tab | 4个（总览/盯盘/对比/批次） | **3个**（每天的情况/关键指标/对比变化） | 名字一看就懂 |
| 上传步骤 | 5步 | **3步** | 不会中途放弃 |
| 首次引导 | 无 | **3步强制引导** | 打开就知道做什么 |
| 家庭共享 | 邀请码+验证+等待确认 | **一键微信分享** | 和拉微信群一样简单 |
| AI入口 | 三层（Tab/悬浮球/页面按钮） | **Tab+悬浮球**（双入口统一面板） | 深度对话用Tab，快速提问用悬浮球 |
| 异常行动 | 3级（偏离百分比） | **2级**（人话描述） | 不用看懂数字 |
| 时间轴节点 | 8种颜色 | **3种颜色**（按严重程度） | 不用记忆图标含义 |
| 每日推送 | 多条分散 | **合并为一条"今日待办"** | 不打扰 |
