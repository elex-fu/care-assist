## 一、设计原则

| 原则 | 含义 | 执行标准 |
|------|------|---------|
| **红绿灯法则** | 健康状态只看颜色 | 绿色=正常、黄色=注意、红色=异常。不需要看懂数字 |
| **一个入口** | 同一功能只有一个面板 | AI的Tab和悬浮球唤起同一面板；上传的"+"和上传Tab唤起同一流程 |
| **人话优先** | 不出现医学术语 | "血压高了一点点"代替"收缩压偏高15%" |
| **自动处理** | 能自动的绝不让用户选 | AI自动匹配成员、自动分类报告、自动推荐行动 |
| **先结论后细节** | 第一眼给答案 | 首页先看状态，详情再看数据 |

## 二、色彩系统

### 2.1 主色调（信任与清晰）

| Token | 色值 | 用途 |
|-------|------|------|
| `--primary-dark` | #2563EB | 主要按钮、激活状态 |
| `--primary-mid` | #3B82F6 | 图标、次级按钮 |
| `--primary-light` | #EFF6FF | 卡片背景、浅色填充、横幅背景 |
| `--primary-fade` | #DBEAFE | 分割线、表单边框、空态插画底色 |

### 2.2 红绿灯状态色（健康状态 — 唯一状态语言）

| Token | 色值 | 用途 |
|-------|------|------|
| `--status-green` | #10B981 | 正常、健康、已完成的疫苗 |
| `--status-green-light` | #ECFDF5 | 正常状态背景 |
| `--status-yellow` | #F59E0B | 注意、轻微异常、即将到期 |
| `--status-yellow-light` | #FEF3C7 | 注意状态背景 |
| `--status-red` | #EF4444 | 严重异常、住院中、逾期 |
| `--status-red-light` | #FEF2F2 | 异常状态背景 |

**后端状态 → 前端颜色映射（唯一映射规则）**：

| 后端状态 | 前端颜色 | 显示文案（示例） |
|----------|----------|------------------|
| `normal` | 🟢 `--status-green` | "健康"/"正常" |
| `low` | 🟡 `--status-yellow` | "偏低"/"低了一些" |
| `high` | 🟡 `--status-yellow` | "偏高"/"高了一点" |
| `critical` | 🔴 `--status-red` | "严重异常"/"尽快就医" |

**关键约定**：
- `low` 和 `high` 在UI层合并为同一颜色（黄灯），但在文案中必须区分方向
- `critical` 除红色外，必须同时显示文字标签，不可仅用颜色传达
- 所有状态显示必须伴随文字，不可仅使用颜色（色盲友好）

### 2.3 AI品牌色

| Token | 色值 | 用途 |
|-------|------|------|
| `--ai-purple` | #8B5CF6 | AI助手标识、悬浮球、消息气泡 |
| `--ai-purple-light` | #F3E8FF | AI消息气泡背景 |
| `--ai-purple-gradient` | #8B5CF6 → #7C3AED | AI悬浮球背景 |

**AI视觉身份规范**：
- **无头像/无拟人形象**：健康是严肃话题，卡通 mascot 会削弱信任感。AI 以抽象 sparkle 图标呈现。
- **sparkle 图标**：24px，紫色(#8B5CF6)，4角星形，用于消息气泡左上角和悬浮球
- **输入中状态**：8px 紫色脉冲圆点，1.5s 呼吸动画，替代 "AI正在输入..." 文字
- **标签文案**："AI说" 或 "健康助手" — 从不使用人名或昵称
- **声音特征**：温暖、直白、有行动建议。不用医学术语，不用过度安慰（"别担心"），而是给出具体下一步。

### 2.4 中性色

| Token | 色值 | 用途 |
|-------|------|------|
| `--text-primary` | #1E293B | 标题、重要正文 |
| `--text-secondary` | #64748B | 次要信息、说明 |
| `--text-tertiary` | #94A3B8 | 占位符、禁用态 |
| `--bg-primary` | #F8FAFC | 页面底色 |
| `--bg-card` | #FFFFFF | 卡片底色 |
| `--divider` | #E2E8F0 | 细分割线 |

### 2.5 色彩对比度合规

| 组合 | 对比度 | WCAG 等级 | 备注 |
|------|--------|-----------|------|
| `--text-primary` on `--bg-card` | 12.6:1 | AAA | 正文 |
| `--text-secondary` on `--bg-card` | 5.7:1 | AA | 次要文字 |
| `--status-green` on `--bg-card` | 3.9:1 | AA Large | 绿色状态文字 → 必须配合图标 |
| `--status-yellow` on `--bg-card` | 2.9:1 | ❌ | 黄色文字禁用 → 黄状态用深色背景配深色文字 |
| `--ai-purple` on `--bg-card` | 5.4:1 | AA | AI紫色 |
| 白色 on `--primary-dark` | 4.5:1 | AA | 主按钮 |

## 四、间距与圆角

### 4.1 间距体系

基础单位：4px

| Token | 值 | 用途 |
|-------|-----|------|
| `--space-xs` | 4px | 图标内边距 |
| `--space-sm` | 8px | 小元素间距 |
| `--space-md` | 16px | 卡片内边距 |
| `--space-lg` | 24px | 卡片外边距 |
| `--space-xl` | 32px | 页面板块间距 |

### 4.2 圆角体系

| 组件 | Token | 值 | 说明 |
|------|-------|-----|------|
| 成员卡片 | `--radius-member-card` | 不对称 20/12/20/12 | 品牌识别元素，主视觉焦点 |
| 指标卡片 | `--radius-card` | 12px | 标准功能卡片 |
| 报告卡片 | `--radius-card` | 12px | 标准功能卡片 |
| 住院批次卡片 | `--radius-card` | 12px | 标准功能卡片 |
| 用药卡片 | `--radius-card` | 12px | 标准功能卡片 |
| AI消息气泡 | `--radius-ai-bubble` | 不对称 12/20/20/12 | 与成员卡片镜像呼应 |
| 用户消息气泡 | `--radius-user-bubble` | 不对称 20/12/12/20 | AI气泡的镜像 |
| 小按钮/标签 | `--radius-sm` | 8px | 小按钮、标签 |
| 主按钮/弹窗 | `--radius-md` | 12px | 主按钮、弹窗 |
| 头像/胶囊 | `--radius-full` | 999px | 全圆 |

**不对称圆角设计原则**：仅在"对话双方"（成员卡片 ↔ AI/用户消息）和主视觉焦点上使用不对称圆角，形成品牌识别。功能卡片保持标准圆角以确保信息密度和可读性。

## 六、组件规范

### 6.1 成员卡片（首页核心组件）

```
尺寸：宽度 = (屏幕宽 - 48) / 2，高度自适应
背景：--bg-card (#FFFFFF)
圆角：--radius-member-card (20/12/20/12)
阴影：0 2px 8px rgba(0,0,0,0.06)
内边距：--space-md (16px)
```

**品牌识别细节**：不对称圆角创造"关怀"的柔软感，区别于普通圆角矩形。视觉上轻盈、有温度，不增加信息密度。

**状态变化**：
- normal → 🟢 健康（文字 `--status-green`）
- low → 🟡 偏低（文字 `--status-yellow`）——只说人话，如"血压低了一点"
- high → 🟡 偏高（文字 `--status-yellow`）——只说人话，如"血压高了一点"
- critical → 🔴 建议复查（文字 `--status-red`）
- 住院中：🔴 住院第5天（文字 `--status-red`）——按业务状态覆盖
- 疫苗快到期：🟡 疫苗快到期（文字 `--status-yellow`）——按业务规则覆盖

### 6.2 AI悬浮球（全局辅助AI入口）

```
位置：右下角，距边缘 20px，距底部 TabBar 20px
尺寸：56px × 56px
背景：线性渐变 --ai-purple → #7C3AED
圆角：--radius-full (999px)
图标：白色 sparkle，28px
阴影：0 4px 16px rgba(139,92,246,0.3)
```

**行为**：
- 点击：展开AI对话浮层（从底部滑出），与底部AI Tab唤起同一面板
- 拖拽：可拖拽到屏幕任意位置（松手后缓动画回边界）
- 手势阈值（防止点击误识别为拖拽）：
  - 触摸移动距离 < 10px 且 按下时间 < 200ms → 识别为点击
  - 触摸移动距离 >= 10px 或 按下时间 >= 200ms → 识别为拖拽
  - 松手时若移动距离 < 10px → 视为点击，触发打开面板

### 6.3 按钮规范

**主按钮**：
```
背景：--primary-dark (#2563EB)
文字：白色，--font-body (16px) 500
高度：48px
圆角：--radius-md (12px)
全宽（减32px边距）
```

**次级按钮**：
```
背景：--bg-card (#FFFFFF)
边框：1px --primary-dark (#2563EB)
文字：--primary-dark (#2563EB)，--font-body (16px) 500
高度：44px
圆角：--radius-md (12px)
```

**文字按钮**：
```
背景：无
文字：--primary-dark (#2563EB)，--font-caption (14px) 400
下划线：无
```

**禁用态**：
```
背景：--divider (#E2E8F0)
文字：--text-tertiary (#94A3B8)
```

### 6.4 底部导航栏（5Tab）

```
高度：49px + 底部安全区34px = 83px
背景：--bg-card (#FFFFFF)，顶部边框 1px --divider (#E2E8F0)

选中态（普通Tab）：图标+文字 --primary-dark (#2563EB)，文字 12px 500
未选中态：图标+文字 --text-tertiary (#94A3B8)，文字 12px 400
AI Tab特殊态：紫色渐变背景 --ai-purple→#7C3AED，白色图标+文字，凸起2px
```

### 6.5 空态组件

```
插画区域：120×120px，SVG矢量插画
标题：18px, 600, --text-primary (#1E293B)，居中
说明：14px, 400, --text-secondary (#64748B)，居中，最多2行
操作按钮：主色按钮（如果有行动）
```

**插画风格规范**：
- 风格：手绘感扁平插画，圆角几何形状，无描边，柔和渐变
- 色彩：以 `--primary-fade` (#DBEAFE) 为底色，搭配状态色点缀（绿/黄/紫），整体温暖不冰冷
- 人物：简化几何人形，无具体五官，用姿态和动作传递情感
- 禁止：3D渲染、写实照片、复杂背景、商业图库感

### 6.6 加载态

**骨架屏**：
```
灰色脉冲动画
background: linear-gradient(90deg, #F1F5F9 25%, #E2E8F0 50%, #F1F5F9 75%)
动画：shimmer, 1.5s infinite
圆角与真实元素一致
```

**全屏Loading**：
```
紫色sparkle旋转动画
文案 + 进度条
```

## 七、响应式适配

| 屏幕类型 | 宽度范围 | 关键适配规则 |
|---------|---------|------------|
| 小屏 (iPhone SE) | ≤375px | 成员卡片内边距降至12px；标题18px；底部Tab图标20px；页边距降至12px |
| 基准 (iPhone 14) | 376-419px | 按设计稿1:1实现 |
| 大屏 (iPhone 15 PM) | ≥420px | 成员卡片最大宽度190px（居中排布）；内容区最大宽度430px居中；两侧留--bg-primary灰边 |

**安全区适配**：
- 顶部：StatusBar 高度动态获取（`wx.getMenuButtonBoundingClientRect`），导航栏内容始终避开胶囊按钮
- 底部：TabBar 高度 = 49px + `safeArea.bottom` 动态计算，iPhone X+ 系列增加 Home Indicator 区域

## 八、无障碍设计

### 8.1 色盲友好

红绿灯是核心状态语言，但约8%男性用户无法区分红绿。因此所有状态标识必须同时提供：

| 状态 | 颜色 | 图标 | 文字标签 | 示例 |
|------|------|------|----------|------|
| 正常 | 🟢 | ✓ | "正常" | 🟢✓ 正常 |
| 注意 | 🟡 | ! | "偏高"/"偏低" | 🟡! 偏高 |
| 异常 | 🔴 | ✕ | "建议复查" | 🔴✕ 建议复查 |

**实现要求**：
- 所有成员卡片、指标卡片、时间轴节点必须同时显示颜色和文字
- 状态Badge不得仅使用颜色，必须包含文字标签
- 图标与颜色绑定：✓=正常、!=注意、✕=异常，形成独立于颜色的第二识别维度

### 8.2 屏幕阅读器适配

- 所有成员卡片：`aria-label="{成员名}，{状态描述}，点击查看详情"`
- 状态Badge：`aria-label="{指标名} {数值}，状态：{正常/偏高/偏低/严重异常}"`
- 趋势图：`aria-label="{指标名}最近{N}次记录，整体趋势{上升/下降/平稳}"`
- 操作按钮：必须包含明确的操作结果描述，如`aria-label="删除爸爸的血压记录"`而非`aria-label="删除"`
- 空态插画：必须设置 `aria-hidden="true"`，由标题和说明文字承担读屏内容

### 8.3 触觉反馈

| 场景 | 反馈类型 | 说明 |
|------|---------|------|
| 上传成功 | light impact | 轻柔确认 |
| 发现异常 | medium impact | 引起注意，不惊吓 |
| 删除确认 | heavy impact | 不可逆操作的物理确认感 |
| 刷新完成 | light impact | 轻微确认 |
| 扫码/识别成功 | light impact + 成功提示音 | 可选，默认关闭 |
| 错误/失败 | medium impact | 与成功区分 |

### 8.4 长辈模式

**核心设计**：长辈模式 = 单成员聚焦视图。首页只显示当前登录 elderly 成员自己的大卡片，去除多成员管理复杂度。

| 功能 | 标准模式 | 长辈模式 | 理由 |
|------|---------|---------|------|
| 首页视图 | 多成员网格 | 单成员大图（全宽，巨大头像+大数字） | 减少认知负荷 |
| 数据范围 | 全家可见 | 仅本人数据 | 保护隐私，减少认知负荷 |
| 基准字体 | 16px | 20px（大）/ 24px（超大） | 易读性 |
| 按钮高度 | 48px | 56px | 易点击 |
| 底部Tab | 5个 | 3个（首页/上传/我的） | 简化导航 |
| AI入口 | Tab+悬浮球 | 仅悬浮球（更大72px） | 减少选择 |
| 输入方式 | 键盘默认 | 语音默认（可切换键盘） | 降低操作门槛 |
| 返回完整模式 | 无 | 设置页底部隐藏"切换回完整家庭模式" | 防止误触 |

## 九、设计决策附录

| 决策 | 结论 | 理由 |
|------|------|------|
| 深色模式 | ❌ 不支持 | 健康应用需要明亮、干净的界面以建立信任感。报告照片和图表在深色背景下可读性下降。 |
| 通知频率 | ✅ 智能打包 | 非紧急通知统一打包为每日8点摘要；仅严重异常（`critical`）立即推送。 |
| AI故障切换 | 🔇 静默切换 | 自动切换到备用服务商，用户无感知。确保核心功能连续性。 |
| 成员卡片圆角 | 不对称 20/12/20/12 | 品牌识别元素，传递"关怀"的柔软感。 |
| AI/用户气泡圆角 | 不对称（镜像） | AI气泡 12/20/20/12，用户气泡 20/12/12/20，形成对话伙伴感。 |
| 空态文案 tone | 温暖、有人情味 | 如"你还没有添加家人呢，快把最爱的他们加进来吧~" |
| 最近查看槽位 | 3个固定 + 横向滚动 | 平衡快捷访问与屏幕空间。无记录时隐藏。 |
| AI空态问候 | 上下文自适应 + 预设兜底 | 有近期活动时显示上下文问候；无数据时显示4个固定预设问题。 |
| 长辈模式 | 单成员聚焦 + 隐藏全模式开关 | 去除多成员复杂度，防止老人误触退出。 |
