## 一、设计概述

### 1.1 设计理念

**「一眼看懂，一步操作」**

所有设计决策以"小白用户零学习成本"为第一优先级。用户不需要理解医学术语、不需要学习复杂的导航逻辑、不需要记忆功能位置。打开即用，看到即懂。

### 1.2 设计原则

| 原则 | 含义 | 执行标准 |
|------|------|---------|
| **红绿灯法则** | 健康状态只看颜色 | 绿色=正常、黄色=注意、红色=异常。不需要看懂数字 |
| **一个入口** | 同一功能只有一个面板 | AI的Tab和悬浮球唤起同一面板；上传的"+"和上传Tab唤起同一流程；同一时间轴视图贯穿各入口 |
| **人话优先** | 不出现医学术语 | "血压高了一点点"代替"收缩压偏高15%" |
| **自动处理** | 能自动的绝不让用户选 | AI自动匹配成员、自动分类报告、自动推荐行动 |
| **先结论后细节** | 第一眼给答案 | 首页先看状态，详情再看数据 |

### 1.3 屏幕基准

```
设计基准：iPhone 14
物理尺寸：390 × 844 pt（逻辑像素）
StatusBar高度：47px（刘海屏）
底部安全区：34px（Home Indicator）
内容安全区：top ≥ 91px, bottom ≥ 83px
```

### 1.4 网格系统

```
基础单位：4px
间距阶梯：4 / 8 / 12 / 16 / 20 / 24 / 32 / 48
所有间距必须是 4px 的整数倍
```

### 1.5 可访问性设计

**色盲友好（红绿色盲覆盖）**：

红绿灯是核心状态语言，但约8%男性用户无法区分红绿。因此所有状态标识必须同时提供：

| 状态 | 颜色 | 图标 | 文字标签 | 示例 |
|------|------|------|----------|------|
| 正常 | 🟢 | ✓ | "正常" | 🟢✓ 正常 |
| 注意 | 🟡 | ! | "偏高"/"偏低" | 🟡! 偏高 |
| 异常 | 🔴 | ✕ | "建议复查" | 🔴✕ 建议复查 |

**实现要求**：
- 所有成员卡片、指标卡片、时间轴节点必须同时显示颜色和文字
- 状态Badge不得仅使用颜色，必须包含文字标签
- 图标与颜色绑定：✓=正常、!=注意、✕=异常，形成独立于颜色的第二识别维度

**长辈模式适配**：
- 字体放大至1.3倍（基准14px→18px，标题20px→26px）
- 按钮高度不低于56px，点击区域不小于48×48pt
- 减少页面信息密度，隐藏儿童专区/住院专区等复杂功能
- 语音输入优先，减少手动输入

| 主色淡 | #DBEAFE | 分割线、表单边框 |

#### 红绿灯（健康状态 — 唯一状态语言）

| 名称 | 色值 | 用途 |
|------|------|------|
| 绿灯 | #10B981 | 正常、健康、已完成的疫苗 |
| 绿灯浅 | #ECFDF5 | 正常状态背景 |
| 黄灯 | #F59E0B | 注意、轻微异常、即将到期 |
| 黄灯浅 | #FEF3C7 | 注意状态背景 |
| 红灯 | #EF4444 | 严重异常、住院中、逾期 |
| 红灯浅 | #FEF2F2 | 异常状态背景 |

**后端状态 → 前端颜色映射（唯一映射规则）**：

| 后端状态 | 前端颜色 | 显示文案（示例） | 说明 |
|----------|----------|------------------|------|
| `normal` | 🟢 绿灯 | "健康"/"正常" | 数值在参考范围内 |
| `low` | 🟡 黄灯 | "偏低"/"低了一些" | 数值低于下限但未达危急 |
| `high` | 🟡 黄灯 | "偏高"/"高了一点" | 数值高于上限但未达危急 |
| `critical` | 🔴 红灯 | "严重异常"/"尽快就医" | 偏离参考范围30%以上或趋势恶化 |

**关键约定**：
- `low` 和 `high` 在UI层合并为同一颜色（黄灯），但在文案中必须区分方向（偏低/偏高）
- `critical` 除红色外，必须同时显示文字标签，不可仅用颜色传达
- 所有状态显示必须伴随文字，不可仅使用颜色（色盲友好）

### 2.5 全局组件

#### 成员卡片（首页核心组件）

```
尺寸：宽度 = (屏幕宽 - 48) / 2，高度自适应
背景：#FFFFFF
圆角：16px
阴影：0 2px 8px rgba(0,0,0,0.06)
内边距：16px

结构：
┌──────────────────┐
│     [头像]        │  ← 48px圆形，居中
│                  │
│     爸爸          │  ← 17px 600 #1E293B，居中
│                  │
│   🟢 血压128/82   │  ← 14px 500，居中，颜色根据状态变化
│     ↓12           │     慢病成员显示关注指标值+趋势
│                  │
│   🟡 疫苗85%      │  ← 儿童成员显示疫苗进度+下次接种
│     5天后到期     │
│                  │
│   🔴 住院第5天    │  ← 住院成员显示住院天数+今日异常数
│     今日2项异常   │
└──────────────────┘

状态变化（按后端→前端映射）：
· normal → 🟢 健康（文字 #10B981）
· low → 🟡 偏低（文字 #F59E0B）——只说人话，如"血压低了一点"
· high → 🟡 偏高（文字 #F59E0B）——只说人话，如"血压高了一点"
· critical → 🔴 建议复查（文字 #EF4444）
· 住院中：🔴 住院第5天（文字 #EF4444）——按业务状态覆盖
· 疫苗快到期：🟡 疫苗快到期（文字 #F59E0B）——按业务规则覆盖
```

#### AI悬浮球（全局辅助AI入口）

```
位置：右下角，距边缘 20px，距底部 TabBar 20px
尺寸：56px × 56px
背景：线性渐变 #8B5CF6 → #7C3AED
圆角：999px（正圆）
图标：白色 sparkle，28px
阴影：0 4px 16px rgba(139,92,246,0.3)

行为：
· 点击：展开AI对话浮层（从底部滑出），与底部AI Tab唤起同一面板
· 新消息：红点角标（16px，#EF4444）
· 拖拽：可拖拽到屏幕任意位置（松手后缓动画回边界）
· 手势阈值（防止点击误识别为拖拽）：
  - 触摸移动距离 < 10px 且 按下时间 < 200ms → 识别为点击，打开AI面板
  - 触摸移动距离 >= 10px 或 按下时间 >= 200ms → 识别为拖拽，跟随手指移动
  - 松手时若移动距离 < 10px → 视为点击，触发打开面板
· 说明：悬浮球与底部AI Tab是同一功能的双入口。Tab适合深度对话，悬浮球适合随时快速提问，二者唤起同一AI对话面板，避免用户困惑。
```

#### 上传按钮（首页快捷上传入口）

```
位置：首页右上角导航栏
尺寸：32px × 32px
图标："+"号，24px，#2563EB
背景：无（纯图标按钮）

行为：
· 点击：唤起相机（默认拍照，长按或点击更多显示"从相册选"）
· 说明：首页快捷上传入口，与底部"上传"Tab功能一致，二者唤起同一上传流程，为用户提供最方便的路径。
```

#### 底部导航栏（5Tab）

```
高度：49px + 底部安全区34px = 83px
背景：#FFFFFF，顶部边框 1px #E2E8F0

┌─────────┬─────────┬─────────┬─────────┬─────────┐
│  首页   │  指标   │   AI    │  上传   │  我的   │
│   🏠    │   📈    │   🤖    │   📤    │   👤    │
│(选中蓝) │(未选中灰)│(紫色渐变)│(未选中灰)│(未选中灰)│
└─────────┴─────────┴─────────┴─────────┴─────────┘

选中态（普通Tab）：图标+文字 #2563EB，文字 12px 500
未选中态：图标+文字 #94A3B8，文字 12px 400
AI Tab特殊态：紫色渐变背景 #8B5CF6→#7C3AED，白色图标+文字，凸起2px
```

**错误类型与对应文案**：

| 错误类型 | 文案 | 行动按钮 |
|----------|------|----------|
| 图片模糊 | "照片有点模糊，再拍一张试试？" | 重新拍照 / 手动输入 |
| 非检验报告 | "这好像不是检验报告？再拍一张试试" | 重新拍照 / 手动输入 |
| 网络中断 | "网络不太稳定，上传失败了" | 重试上传 / 保留草稿 |
| AI服务繁忙 | "AI暂时忙不过来，稍后再试？" | 保留原图，稍后自动重试 / 手动输入 |
| 识别结果为空 | "没从照片里找到检验指标" | 重新拍照 / 手动输入 |

**保留原图逻辑**：
- 点击"保留原图，稍后再试"：报告存入 `reports` 表，状态 `ocr_status='failed'`，原图保留
- 用户在时间轴看到该记录，标记为"🟡 待识别"，可点击重新触发OCR
- 后台每小时自动重试所有 `failed` 状态的报告（最多再试3次）

│ └────────────────────────┘  │

│                              │
│ 收缩压：                     │
│ ┌────────┐  mmHg           │
│ │  128   │                 │  ← 数值输入框
│ └────────┘                 │
│                              │

│ ┌────────┐  mmHg           │
│ │  82    │                 │
│ └────────┘                 │

#### 设计要点

- **常见指标选择器**：提供 8 个最常用指标快捷按钮，减少用户输入负担
- **自动带出单位和参考范围**：选择指标后，单位（mmHg、mmol/L 等）和参考范围自动填充，用户只需填数值
- **搜索支持模糊匹配**：输入"血红"可匹配到"血红蛋白"
- **日期默认今天**：用户只需改数值，不改日期
- **多指标连续录入**：保存后提示"继续录入下一个指标？"，减少返回再进入的操作
