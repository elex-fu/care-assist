## 一、设计概述

### 1.1 设计理念

**「一眼看懂，一步操作」**

所有设计决策以"小白用户零学习成本"为第一优先级。用户不需要理解医学术语、不需要学习复杂的导航逻辑、不需要记忆功能位置。打开即用，看到即懂。

### 1.2 设计原则

| 原则 | 含义 | 执行标准 |
|------|------|---------|
| **红绿灯法则** | 健康状态只看颜色 | 绿色=正常、黄色=注意、红色=异常。不需要看懂数字 |
| **一个入口** | 同一功能只有一个面板 | AI的Tab和悬浮球唤起同一面板；上传的"+"和上传Tab唤起同一流程；同一时间轴视图贯穿各入口 |
| **人话优先** | 不出现医学术语 | "血压高了一点点"代替"收缩压偏高15%" |
| **自动处理** | 能自动的绝不让用户选 | AI自动匹配成员、自动分类报告、自动推荐行动 |
| **先结论后细节** | 第一眼给答案 | 首页先看状态，详情再看数据 |

### 1.3 用户旅程与情感设计

**设计目标**：用户在不同阶段感受到的情绪，必须有对应的设计支持。

| 步骤 | 用户在做什么 | 用户感受 | 设计如何支持 |
|------|------------|---------|------------|
| 1 | 第一次打开小程序 | 好奇，有点不知道怎么开始 | P10 欢迎页用温暖插画+"拍一拍，管全家"降低门槛 |
| 2 | 添加第一位家人 | 认真，有责任感 | 关系预设胶囊（爸爸/妈妈/孩子），不用想怎么填 |
| 3 | 拍第一张报告 | 不确定"这能识别出来吗？" | 大相机按钮+没有报告也可以手动录入，零压力 |
| 4 | 看到第一个异常结果 | 有点担心 | AI用"高了一点"而不是"偏高15%"，温和但有行动按钮 |
| 5 | 收到第一个提醒 | 被照顾到的感觉 | 提醒文案具体到人和事："爸爸该复查血压啦" |
| 6 | 长期使用，管理慢病 | 有掌控感 | 趋势图+用药打卡，看得见自己的坚持 |
| 7 | 家人生病住院 | 紧张，需要快速了解状况 | 住院页直接显示今日异常数，不用翻找 |
| 8 | 几年后回顾全家健康 | 温暖的记忆 | 时间轴是全家健康的回忆录，数据导出留下纪念 |

**关键时刻设计**：
- **危机时刻**（发现严重异常）：🔴颜色传递紧迫感，但AI消息和行动按钮立即给用户"我知道该做什么"的掌控感。绝不能只给红色不给出路。
- **空白时刻**（没有任何记录时）：用温暖、有人情味的文案代替冷冰冰的功能说明。例如："你还没有添加家人呢，快把最爱的他们加进来吧~" 而不是 "添加第一位家人，开启全家健康管理"。
- **成就时刻**（连续记录、按时复查）：用微动画和小庆祝（如"已连续记录7天 🎉"）强化正向行为。

### 1.4 屏幕基准

```
StatusBar高度：47px（刘海屏）

```

### 1.4 网格系统

```

所有间距必须是 4px 的整数倍
```

### 1.5 可访问性设计

**色盲友好（红绿色盲覆盖）**：

红绿灯是核心状态语言，但约8%男性用户无法区分红绿。因此所有状态标识必须同时提供：

| 状态 | 颜色 | 图标 | 文字标签 | 示例 |
|------|------|------|----------|------|
| 正常 | 🟢 | ✓ | "正常" | 🟢✓ 正常 |
| 注意 | 🟡 | ! | "偏高"/"偏低" | 🟡! 偏高 |
| 异常 | 🔴 | ✕ | "建议复查" | 🔴✕ 建议复查 |

**实现要求**：
- 所有成员卡片、指标卡片、时间轴节点必须同时显示颜色和文字
- 状态Badge不得仅使用颜色，必须包含文字标签
- 图标与颜色绑定：✓=正常、!=注意、✕=异常，形成独立于颜色的第二识别维度

**长辈模式适配**：
- 字体放大至1.3倍（基准14px→18px，标题20px→26px）
- 按钮高度不低于56px，点击区域不小于48×48pt
- 减少页面信息密度，隐藏儿童专区/住院专区等复杂功能
- 语音输入优先，减少手动输入

| 组件 | 圆角 | 说明 |

**不对称圆角设计原则**：仅在"对话双方"（成员卡片 ↔ AI/用户消息）和主视觉焦点上使用不对称圆角，形成品牌识别。功能卡片保持标准圆角以确保信息密度和可读性。

#### 红绿灯（健康状态 — 唯一状态语言）

| 名称 | 色值 | 用途 |
|------|------|------|
| 绿灯 | #10B981 | 正常、健康、已完成的疫苗 |
| 绿灯浅 | #ECFDF5 | 正常状态背景 |
| 黄灯 | #F59E0B | 注意、轻微异常、即将到期 |
| 黄灯浅 | #FEF3C7 | 注意状态背景 |
| 红灯 | #EF4444 | 严重异常、住院中、逾期 |
| 红灯浅 | #FEF2F2 | 异常状态背景 |

**后端状态 → 前端颜色映射（唯一映射规则）**：

| 后端状态 | 前端颜色 | 显示文案（示例） | 说明 |
|----------|----------|------------------|------|
| `normal` | 🟢 绿灯 | "健康"/"正常" | 数值在参考范围内 |
| `low` | 🟡 黄灯 | "偏低"/"低了一些" | 数值低于下限但未达危急 |
| `high` | 🟡 黄灯 | "偏高"/"高了一点" | 数值高于上限但未达危急 |
| `critical` | 🔴 红灯 | "严重异常"/"尽快就医" | 偏离参考范围30%以上或趋势恶化 |

**关键约定**：
- `low` 和 `high` 在UI层合并为同一颜色（黄灯），但在文案中必须区分方向（偏低/偏高）
- `critical` 除红色外，必须同时显示文字标签，不可仅用颜色传达
- 所有状态显示必须伴随文字，不可仅使用颜色（色盲友好）

#### AI品牌色

| 名称 | 色值 | 用途 |
|------|------|------|
| AI紫 | #8B5CF6 | AI助手标识、悬浮球 |
| AI紫浅 | #F3E8FF | AI消息气泡背景 |
| AI紫渐变 | #8B5CF6 → #7C3AED | AI悬浮球背景 |

**AI视觉身份规范**：
- **无头像/无拟人形象**：健康是严肃话题，卡通 mascot 会削弱信任感。AI 以抽象 sparkle 图标呈现。
- **sparkle 图标**：24px，紫色(#8B5CF6)，4角星形，用于消息气泡左上角和悬浮球
- **输入中状态**：8px 紫色脉冲圆点，1.5s 呼吸动画，替代 "AI正在输入..." 文字
- **标签文案**："AI说" 或 "健康助手" — 从不使用人名或昵称
- **声音特征**：温暖、直白、有行动建议。不用医学术语，不用过度安慰（"别担心"），而是给出具体下一步。

### 2.5 全局组件

#### 成员卡片（首页核心组件）

```
尺寸：宽度 = (屏幕宽 - 48) / 2，高度自适应
背景：#FFFFFF
圆角：不对称设计 — 左上20px / 右上12px / 右下20px / 左下12px
阴影：0 2px 8px rgba(0,0,0,0.06)
内边距：16px

**品牌识别细节**：不对称圆角创造"关怀"的柔软感，区别于普通圆角矩形。视觉上轻盈、有温度，不增加信息密度。

结构：
┌──────────────────┐
│     [头像]        │  ← 48px圆形，居中
│                  │
│     爸爸          │  ← 17px 600 #1E293B，居中
│                  │
│   🟢 血压128/82   │  ← 14px 500，居中，颜色根据状态变化
│     ↓12           │     慢病成员显示关注指标值+趋势
│                  │
│   🟡 疫苗85%      │  ← 儿童成员显示疫苗进度+下次接种
│     5天后到期     │
│                  │
│   🔴 住院第5天    │  ← 住院成员显示住院天数+今日异常数
│     今日2项异常   │
└──────────────────┘

状态变化（按后端→前端映射）：
· normal → 🟢 健康（文字 #10B981）
· low → 🟡 偏低（文字 #F59E0B）——只说人话，如"血压低了一点"
· high → 🟡 偏高（文字 #F59E0B）——只说人话，如"血压高了一点"
· critical → 🔴 建议复查（文字 #EF4444）
· 住院中：🔴 住院第5天（文字 #EF4444）——按业务状态覆盖
· 疫苗快到期：🟡 疫苗快到期（文字 #F59E0B）——按业务规则覆盖
```

#### AI悬浮球（全局辅助AI入口）

```
位置：右下角，距边缘 20px，距底部 TabBar 20px
尺寸：56px × 56px
背景：线性渐变 #8B5CF6 → #7C3AED
圆角：999px（正圆）
图标：白色 sparkle，28px
阴影：0 4px 16px rgba(139,92,246,0.3)

行为：
· 点击：展开AI对话浮层（从底部滑出），与底部AI Tab唤起同一面板
· 新消息：红点角标（16px，#EF4444）
· 拖拽：可拖拽到屏幕任意位置（松手后缓动画回边界）
· 手势阈值（防止点击误识别为拖拽）：
  - 触摸移动距离 < 10px 且 按下时间 < 200ms → 识别为点击，打开AI面板
  - 触摸移动距离 >= 10px 或 按下时间 >= 200ms → 识别为拖拽，跟随手指移动
  - 松手时若移动距离 < 10px → 视为点击，触发打开面板
· 说明：悬浮球与底部AI Tab是同一功能的双入口。Tab适合深度对话，悬浮球适合随时快速提问，二者唤起同一AI对话面板，避免用户困惑。
```

### 2.6 响应式适配

微信小程序运行在各种尺寸的手机上，必须保证从 iPhone SE 到 iPhone 15 Pro Max 的体验一致。

| 屏幕类型 | 宽度范围 | 关键适配规则 |
|---------|---------|------------|
| 小屏 (iPhone SE) | ≤375px | 成员卡片内边距降至12px；标题18px；底部Tab图标20px；页边距降至12px |
| 基准 (iPhone 14) | 376-419px | 按设计稿1:1实现 |
| 大屏 (iPhone 15 PM) | ≥420px | 成员卡片最大宽度190px（居中排布）；内容区最大宽度430px居中；两侧留#F8FAFC灰边 |
| 平板 (iPad) | ≥768px | 强制竖屏，内容区最大宽度430px居中，两侧大面积留白，不拉伸 |

**安全区适配**：
- 顶部：StatusBar 高度动态获取（wx.getMenuButtonBoundingClientRect），导航栏内容始终避开胶囊按钮
- 底部：TabBar 高度 = 49px + safeArea.bottom 动态计算，iPhone X+ 系列增加 Home Indicator 区域

### 2.7 无障碍与触觉设计

**色彩对比度**（WCAG 2.1 合规）：
| 组合 | 对比度 | 等级 | 备注 |
|------|--------|------|------|
| #1E293B on #FFFFFF | 12.6:1 | AAA | 正文 |
| #64748B on #FFFFFF | 5.7:1 | AA | 次要文字 |
| #10B981 on #FFFFFF | 3.9:1 | AA Large | 绿色状态文字 → 必须配合图标，不可单独使用 |
| #F59E0B on #FFFFFF | 2.9:1 | ❌ | 黄色文字禁用 → 黄状态用深色背景(#FEF3C7)配深色文字(#92400E) |
| #8B5CF6 on #FFFFFF | 5.4:1 | AA | AI紫色 |
| 白色 on #2563EB | 4.5:1 | AA | 主按钮 |

**屏幕阅读器适配**：
- 所有成员卡片：`aria-label="{成员名}，{状态描述}，点击查看详情"`
- 状态Badge：`aria-label="{指标名} {数值}，状态：{正常/偏高/偏低/严重异常}"`
- 趋势图：`aria-label="{指标名}最近{N}次记录，整体趋势{上升/下降/平稳}"`
- 操作按钮：必须包含明确的操作结果描述，如`aria-label="删除爸爸的血压记录"`而非`aria-label="删除"`
- 空态插画：必须设置 `aria-hidden="true"`，由标题和说明文字承担读屏内容

**触觉反馈映射**：
| 场景 | 反馈类型 | 说明 |
|------|---------|------|
| 上传成功 | light impact | 轻柔确认 |
| 发现异常 | medium impact | 引起注意，不惊吓 |
| 删除确认 | heavy impact | 不可逆操作的物理确认感 |
| 刷新完成 | light impact | 轻微确认 |
| 扫码/识别成功 | light impact + 成功提示音 | 可选，默认关闭 |
| 错误/失败 | medium impact | 与成功区分 |

## 三、页面设计

### 页面导航总览

**错误类型与对应文案**：

| 错误类型 | 文案 | 行动按钮 |
|----------|------|----------|
| 图片模糊 | "照片有点模糊，再拍一张试试？" | 重新拍照 / 手动输入 |

- 用户在时间轴看到该记录，标记为"🟡 待识别"，可点击重新触发OCR
- 后台每小时自动重试所有 `failed` 状态的报告（最多再试3次）

#### 长辈模式下的UI变化

**核心设计**：长辈模式 = 单成员聚焦视图。首页只显示当前登录 elderly 成员自己的大卡片，去除多成员管理复杂度。

| 元素 | 标准模式 | 长辈模式 |
|------|----------|----------|
| 首页视图 | 多成员网格 | 单成员大图（全宽，巨大头像+大数字） |

| 底部Tab | 5个 | 3个（首页/上传/我的） |
| AI入口 | Tab+悬浮球 | 仅悬浮球（更大72px） |
| 首页功能 | 全部显示 | 隐藏儿童/住院/家人管理入口 |
| 输入方式 | 键盘默认 | 语音默认（可切换键盘） |
| 返回完整模式 | 无 | 设置页底部隐藏"切换回完整家庭模式" |

**模式切换规则**：

- 关闭长辈模式：恢复标准5Tab首页，显示全部家庭成员

- 家庭管理员可在"我的"中看到"为{成员名}开启长辈模式"的快捷入口

## 附录：已解决的设计决策

本附录记录设计评审过程中明确的设计决策，供开发实施时参考。

| 决策 | 选项 | 理由 |
|------|------|------|
| **深色模式** | ❌ 不支持 | 健康应用需要明亮、干净的界面以建立信任感。报告照片和图表在深色背景下可读性下降。 |
| **通知频率** | ✅ 智能打包 | 非紧急通知统一打包为每日8点摘要；仅严重异常（`critical`）立即推送。避免消息轰炸导致用户关闭全部通知。 |
| **AI故障切换** | 🔇 静默切换 | 当主AI服务商不可用时，自动切换到备用服务商，用户无感知。确保核心功能连续性。健康建议质量由Provider抽象层保证一致性。 |
| **最近查看槽位** | 3个固定 + 横向滚动 | 平衡快捷访问与屏幕空间。无记录时隐藏区域，保持首屏整洁。 |
| **AI空态问候** | 上下文自适应 + 预设兜底 | 有近期健康活动时显示上下文相关问候；无数据时显示4个固定预设问题。 |
| **空态文案 tone** | 温暖、有人情味 | 例如"你还没有添加家人呢，快把最爱的他们加进来吧~"，而非冷冰冰的功能说明。 |
| **成员卡片圆角** | 不对称 20/12/20/12 | 品牌识别元素，传递"关怀"的柔软感。AI气泡和用户气泡使用镜像不对称圆角。 |
| **长辈模式** | 单成员聚焦 + 隐藏全模式开关 | 首页仅显示 elderly 成员自己的大卡片，去除多成员复杂度。设置页底部隐藏"切换回完整家庭模式"按钮，防止误触。 |

## NOT in scope（本轮评审明确排除）

以下设计方向经过讨论后决定**不在当前版本实施**：

1. **全功能深色模式** — 成本收益比低，健康类应用浅色背景更利于建立信任。
2. **自定义主题色** — 用户不可更改主色调。品牌一致性优先于个性化。
3. **3D/动画插画** — 空态插画保持手绘扁平风格，不引入3D渲染或Lottie复杂动画。
4. **实时多人协作编辑** — 家庭成员数据共享采用"查看+管理员编辑"模式，不实施类似Google Docs的实时协作。
5. **AR/摄像头测量** — 不通过摄像头测量心率、血压等生理指标，避免医疗合规风险。
6. **社交分享（朋友圈/微信好友）** — 健康数据隐私敏感，不支持直接分享到社交网络。仅支持家庭内共享和PDF导出。
7. **语音助手唤醒词** — 不支持"Hey AI"等唤醒词，仅支持点击/悬浮球唤起。降低误触发和隐私风险。
